◆ Update()

void PlayerDefense.Update ( )
inlineprivate

Update() — Se ejecuta cada frame. Lee el input de bloqueo y compara con el estado anterior (isBlocking). Solo ejecuta la lógica de cambio de estado cuando el input cambia, evitando llamadas repetidas a EnergySystem y al Animator cada frame.

CUÁNDO SE EJECUTA: Cada frame durante la partida. QUÉ CONTROLA: Detección de cambio de estado de bloqueo, ganancia de energía al inicio, animación y hitbox.

FLUJO DE CAMBIO DE ESTADO: Input presionado (isBlocking = false → true): → EnergySystem.GainEnergyFromBlock() (solo una vez) → Animator.SetBool("isBlocking", true) → blockHitbox.SetActive(true) Input soltado (isBlocking = true → false): → Animator.SetBool("isBlocking", false) → blockHitbox.SetActive(false)
